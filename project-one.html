GEMINI

PROJECT OVERVIEW

Gemini is a fictional local fashion brand which targets mainly in gen Z and millennials user having a minimal and trendy lifestyle.

The project is to create an interactive prototype for Gemini's website and application.

In this phase, I did some online researches and gathered information to get to know the business, industry and competitors available in the market. Logo and brand style tile also were created for the sake of building interface design consistently in the next phases.

üë§ user ANALYSIS

A User scenario is developed to paint a picture of what the site/app will look like or function like and how users might act to achieve Gemini website/app's goal which is mainly focusing on user online-shopping experience such as: browsing items, search, check-out process.

‚úçüèª ideate

A sitemap is created to show how the website/app content, any specific navigation elements, site or app's labels should be organized and connected with each other.

A User flow of purchasing a product is built to take a user from their entry point through a set of steps towards a successful outcome and final action. By using Flowchart, it helps clarify these difficult processes or procedures to develop Wireframe and further prototype more efficiently.

üìå prototype

Based on sitemap and flowchart, a low-fidelity wireframe is created at the first stage of interface design process to help building overall structure of GEMINI website/app, including: main screen or homepage, along with all screens that demonstrate the key interactions and steps of interface such as: searching or catalogue browsing functionality, bookmarking items into wish-list site, completed purchase and check-out process, login and user settings interfaces.

After finalizing low-fidelity wireframe, the high-fidelity version and interactive prototype are developed and populated with detailed content and visuals of what a real finished product's interface would look like.

üí° understand

Using Userberry/Maze to conduct these usability tests. The high-level purpose of this user testing is to evaluate and validate GEMINI's website/app interface design by letting potential users interact and perform some given tasks on the prototype. After user testing, a report is conducted to point out whether users can complete given tasks successfully or not, or to identify how long a user takes to accomplish each specific task. Then, applying appropriate revisions on the prototype to solve all user's pain points, usability problems and improve user's satisfaction.
